# Eclipse Vert.x Style Community Contributing Guide

This project follows the contribution process used across the Vert.x ecosystem.

## Logging Issues

Log an issue for bugs, regressions, or feature requests. For usage questions, prefer community channels.

## Contributions

- Non-trivial changes should go through pull requests.
- Pull requests require review before merge.
- Keep pull requests focused and include tests for behavior changes.
- Sign off commits with:

```bash
git commit -s -m "Your message"
```

## Copyright headers

Source files should include a legal header that references Apache License 2.0.

## Contributor expectations

- Be respectful and constructive in review discussions.
- Prefer incremental, reviewable changes over large opaque rewrites.
- Keep documentation updated alongside code changes.
